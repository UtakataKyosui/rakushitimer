# CI 設計ドキュメント

**日付**: 2026-03-01
**対象**: rakushitimer (Tauri v2 + React + Android)

---

## 目的

- PR・push 時に自動でテスト・型チェック・ビルド検証を行い、壊れたコードの混入を防ぐ
- タグ push 時に Android デバッグ APK を自動生成し、GitHub Releases に添付する

---

## ワークフロー構成

### 1. `ci.yml` — PR・push チェック

**トリガー**
- `main` ブランチへの push
- 全ブランチへの pull_request

**ジョブ: `frontend`** (ubuntu-latest)
- pnpm セットアップ
- `pnpm install --frozen-lockfile`
- `pnpm test --run`（Vitest）
- `pnpm build`（tsc + vite build）

**ジョブ: `rust`** (ubuntu-latest)
- Rust ツールチェーンセットアップ（stable）
- Rust ビルドキャッシュ（Swatinem/rust-cache）
- `cargo test`（src-tauri 内）

### 2. `release-android.yml` — Android APK ビルド

**トリガー**
- `v*` タグ push（例: `v0.1.0`）

**ジョブ: `build-android`** (ubuntu-latest)

| ステップ | 内容 |
|----------|------|
| JDK 17 セットアップ | `actions/setup-java@v4` (temurin) |
| Android SDK セットアップ | `android-actions/setup-android@v3` |
| Rust + Android ターゲット追加 | aarch64 / armv7 / i686 / x86_64 |
| pnpm セットアップ | `pnpm/action-setup@v4` |
| `pnpm install --frozen-lockfile` | 依存関係インストール |
| `pnpm tauri android build` | デバッグ APK 生成 |
| GitHub Releases に APK アップロード | `softprops/action-gh-release@v2` |

---

## 技術的決定事項

- **APK 署名**: なし（デバッグビルド）。ストア公開は行わないため不要
- **Rust キャッシュ**: `Swatinem/rust-cache` で `src-tauri` を対象にキャッシュし、CI 時間を短縮
- **pnpm**: `pnpm/action-setup` で指定、`package.json` の `packageManager` フィールドを参照
- **Android NDK**: `setup-android` が自動でインストール
