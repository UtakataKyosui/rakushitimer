> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** GitHub Actions で PR/push チェックと Android APK リリースビルドの2本の CI ワークフローを追加する

**Architecture:** `.github/workflows/ci.yml`（frontend + rust ジョブ）と `.github/workflows/release-android.yml`（Android APK ビルド + GitHub Releases アップロード）の2ファイル構成

## Implementation Steps

### Task 1: ci.yml を作成する（frontend + rust チェック）

**Files:**
- Create: `.github/workflows/ci.yml`

**Step 1: `.github/workflows/` ディレクトリを作成**

```bash
mkdir -p .github/workflows
```

**Step 2: `ci.yml` を作成**

`.github/workflows/ci.yml` に以下を書く：

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Test
        run: pnpm test --run

      - name: Build
        run: pnpm build

  rust:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install system dependencies
        run: |
          sudo apt-get update
          sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf

      - uses: dtolnay/rust-toolchain@stable

      - uses: Swatinem/rust-cache@v2
        with:
          workspaces: src-tauri

      - name: Test
        working-directory: src-tauri
        run: cargo test
```

**Step 3: 構文を目視確認**

`on`, `jobs`, `steps` のインデントが正しいか確認する（YAML はインデントに敏感）

**Step 4: コミット**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add CI workflow for frontend and rust checks"
```

### Task 2: release-android.yml を作成する

**Files:**
- Create: `.github/workflows/release-android.yml`

**Step 1: `release-android.yml` を作成**

`.github/workflows/release-android.yml` に以下を書く：

```yaml
name: Release Android

on:
  push:
    tags:
      - 'v*'

jobs:
  build-android:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17

      - name: Setup Android SDK
        uses: android-actions/setup-android@v3

      - name: Install Android NDK
        run: sdkmanager "ndk;27.2.12479018"

      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: aarch64-linux-android,armv7-linux-androideabi,i686-linux-android,x86_64-linux-android

      - uses: Swatinem/rust-cache@v2
        with:
          workspaces: src-tauri

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build Android APK
        run: pnpm tauri android build --debug
        env:
          NDK_HOME: ${{ env.ANDROID_NDK_HOME }}

      - name: Upload APK to GitHub Releases
        uses: softprops/action-gh-release@v2
        with:
          files: src-tauri/gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk
          tag_name: ${{ github.ref_name }}
```

**Step 2: APK 出力パスを確認**

Tauri v2 の Android デバッグビルドの出力先はこのパスになる：
```
src-tauri/gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk
```

既にローカルでビルド済みであれば `ls src-tauri/gen/android/app/build/outputs/apk/` で確認できる。
未ビルドであれば実際の CI 実行時に確認する。

**Step 3: コミット**

```bash
git add .github/workflows/release-android.yml
git commit -m "ci: add Android APK release workflow"
```

## Notes

- `rust` ジョブの `sudo apt-get install` は Tauri のデスクトップビルドに必要なシステム依存（`libwebkit2gtk-4.1-dev` 等）。`cargo test` のみでも必要になる場合がある
- NDK バージョン `27.2.12479018` は Tauri v2 Android の推奨版
- `softprops/action-gh-release@v2` はリポジトリに `GITHUB_TOKEN` の write 権限が必要（デフォルトで付与されている）
